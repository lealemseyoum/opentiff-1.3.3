Introduction

OpenTiff is an object-oriented interface to TIFF formated image files. Unlike other Tiff toolkits, it allows an arbitrary set of tags to be defined and used in a Tiff file.

Toolkit features

  • Source code available under Open Source license enables users to contribute to the toolkit
  • Open source license allows for toolkit usage in commercial applications
  • Provide low level strip-based and tile-based image support.
  • Provide high level RGB image support without regard for the underlining data organization, color space and compression scheme.
  • Support non-compression image data storage and Jpeg compression.
  • Provide an extensible framework for integrating other compression schemes.
  • Capable of reading/writing tags defined in TIFF 6.0 and any undefined tags.
  • Capable of reading/writing sub IFD defined in TIFF 6.0 and any undefined sub IFD.

Build Note

Building on Windows NT/98/2000/XP

Building opentiff requires:

  • NT 3.51 or higher, 95 or 98, 2000, XP
  • VC6 with SP3, VC7, or VC7.1

To build opentiff from its source , you will need to open the workspaces containing the projects. For VC6, they are:

  • projects\Win32\Vc6.0\BuildAllStatic.dsw: for building static libraries and its test program.
  • projects\Win32\Vc6.0\BuildAllDynamic.dsw: for building dlls and its test program.

For VC7 and VC7.1, they are:

  • projects\Win32\VC7.0\BuildAllStatic.sln: for building static libraries and its test program.
  • projects\Win32\VC7.0\BuildAllDynamic.sln: for building dlls and its test program.

If you want to include the opentiff project into your application's workspace, you need to pick up the following projects. For VC6:

  • projects\Win32\Vc6.0\EkTiff.dsp: for building static libraries.
  • projects\Win32\Vc6.0\EkTiffi.dsp: for building dlls.

For VC7 and VC7.1:

  • projects\Win32\VC7.0\EkTiff.vcproj: for building static libraries.
  • projects\Win32\VC7.0\EkTiffi.vcproj: for building dlls.

Building on Unix/Linux

For building opentiff, the following compilers and tools have been tested and are supported:

  • GNU g++ 2.95.3, or greater

The steps are:

  • cd to the root directory of opentiff.
  • type ./configure and then ./make

Building on Mac OS

Building opentiff requires:

  • Mac OS 9 or higher
  • CodeWarrior 4.1.0.3 or higher

To build opentiff library from its source, open the following project with CodeWarrior:

projects/MacOS/CodeWarrior/EkTiff.mcp.
